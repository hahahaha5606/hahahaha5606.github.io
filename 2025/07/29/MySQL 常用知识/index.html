1.简介

MySQL 是一种关系型数据库管理系统软件，用于管理数据库 (DataBase)。

关系型数据库管理系统采用关系模型组织数据，借助于集合、代数等数学概念和方法来处理数据，通过二维表来表示数据之间的联系（二维表的每一行表示一条数据记录，每一列表示一个字段即记录属性），不同的表之间通过关联字段来建立联系。常见的软件有：MySQL、Oracle、PostgreSQL等。

非关系型数据库管理系统是对关系型数据库管理系统的补充和扩展，是为适应互联网时代大量的数据而产生的。常见的软件有：Redis、mongoDB、neo4j、cassandra、CouchDB等。

SQL（Structured Query Language，结构化查询语言）是一种用于操作关系型数据库的语言。关系型数据库中，数据一般以表的形式存储，SQL 可以用于操作表中的数据。SQL 语句不分大小写，但是为提高可读性，一般把关键字大写，而把表名、列名和其他名称小写。注意，SQL 的语句末尾需要加分号！

按照功能不同，可以把 SQL 语句分为以下四类：

类型 说明
DDL（Data Definition Language，数据定义语言） 关键字有 Create、DROP、ALTER、TRUNCATE
DML（Data Manipulation Language，数据操作语言） 关键字有 INSERT、UPDATE、DELETE、CALL
DQL（Data Query Language，数据查询语言） 关键字有 SELECT、WHERE
DDL（Data Control Language，数据控制语言） 定义数据库的访问权限和安全级别，以及创建用户等。关键字有 GRANT、REVOKE 等。
TCL（Transaction Control Language，事务控制语言） 相关关键字包括：COMMIT、ROLLBACK等

SQL 语句的注释方法主要有三种：双减号 -- (前后都要用空格隔开) 或 # 号表示单行注释；/* */ 则表示多行注释。

注：abs()、ceil()、floor()、round() 是 标量函数（逐行计算），不支持 OVER() 子句（窗口函数语法）。

窗口函数是一种对结果集中的行进行分组聚合、排名、累计计算和移动平均等操作的函数，而不改变行数（与 GROUP BY 不同）。OVER()子句是SQL中窗口函数的一部分。它用于将函数应用于表达式的一组行，并且可以根据特定的排序规则进行分区，以便在每个分区中进行计算。

OVER()子句由两部分组成：子句和窗口定义。子句定义了要计算的函数，而窗口定义定义了计算的行集，其基本语法为：

  • 函数名(表达式)：表示在窗口中使用或计算的函数，如 COUNT、SUM等聚合函数；
  • <分组列> ：表示要分区的属性列；
  • <排序列>： 表示根据哪列进行排序；
  • [滑动范围]：如
    • ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING 将滑动范围扩展为整个分区。
    • ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW 范围为第一行到当前行。
    • ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING 范围为当前行的前1行 + 后1行。
    • RANGE BETWEEN INTERVAL 7 DAY PRECEDING AND CURRENT ROW 时间滑动窗口。

字符串操作

1.判断子串

当查找 A 字符串是否在 B 字符串中（A 是 B 的子串）时，可以使用关键字 locate 、position 和 instr：

  • locate 判断是否包含某字符串 如：select * from user where locate('李',user_name)>0（如果不包含，返回0）；
  • position 判断是否包含某字符串 如：select * from user where position('李' in user_name)>0（如果不包含，返回0）；
  • instr 判断是否包含某字符串 如：select * from user where instr(user_name,'李')>0（如果不包含，返回0）。

此外，FIND_IN_SET(str,strlist) 可以用于查询字段(strlist) 中是否包含(str)的结果。FIND_IN_SET 函数用于在以逗号分隔的字符串列表中查找指定字符串的位置。它会返回指定字符串在列表中的位置，如果未找到则返回 0。 如下：select * from user where find_in_set ('刘备',user_name)>0

9.表关联

表关联用于查询多个表中的数据，关联的表之间必须有相同的字段。一般使用表的主键和外键相关联。表关联分为以下几个类型：

  • 内连接（INNER JOIN）：只返回两个表中都有的数据。
  • 左连接（LEFT JOIN）：返回左表中所有和右表匹配的数据，右表中没有的数据用 NULL 填充。
  • 右连接（RIGHT JOIN）：返回右表中所有和左表匹配的数据，左表中没有的数据用 NULL 填充。
  • 全连接（FULL JOIN）：只要数据在其中一个表存在，就返回，其他表中没有的数据用 NULL 填充。

需要注意“笛卡尔积问题”，即如果条件选择不正确（如，不筛选 SELECT * from player, equip），则结果可能会出现"一对多"（表 player 中的一条记录对应 equip 中的多条记录）的问题。

12.事务

  • ROLLBACK 事务回滚（回滚到 BEGIN 语句执行前）

13.锁

MySQL 中的锁有表级锁和行级锁。

可以并发读，但是不能并发写，读锁期间，没释放锁之前不能进行写操作。

使用场景：读取结果集的最新版本，同时防止其他事务产生更新该结果集。主要用在需要数据依存关系时确认某行记录是否存在，并确保没有人对这个记录进行UPDATE 或者 DELETE 操作。

只有锁表的用户可以进行读写操作，其他用户既不能读，也不能写。

复合增长率

复合增长率是第N期的数据除以第1期的基准数据，然后开N-1次方再减去1得到的结果。

假如2018年的产品销售额为10000，2019年的产品销售额为12500，2020年的产品销售额为15000。那么这两年的复合增长率的计算方式如下： \[ \left(\frac{15000}{10000}\right)^{\frac{1}{2}}-1=\sqrt\frac{15000}{10000}-1=22.47\% \] 以年度为单位计算的复合增长率被称为年均复合增长率，以月度为单位计算的复合增长率被称为月均复合增长率（即环比增长率）。以下查询统计了自2018年1月以来不同产品的月均销售额复合增长率：
