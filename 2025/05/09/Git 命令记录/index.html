#### 版本控制系统

目前流行的版本控制系统可分为集中式（如SVN、CVS）和分布式（Git、Mercurial）两种。

集中式版本控制系统：所有文件及其版本保存在服务器（Server）上，每个人电脑（Client）上只保存副本。使用时个人从服务器上下载文件的最新版本，修改后上传回服务器。优点是使用简单；缺点是“单点故障问题”，即如果服务器出现故障或网络连接出现问题，那么工作将被迫暂停直到问题解决。

分布式版本控制系统：个人电脑和服务器上都存有完整的版本库，不存在“单点故障问题”，支持离线工作。如果需要将文件分享给他人，只需同步仓库即可。

注：常见的代码托管平台有Github (开源，国内连接不稳定)、码云 Gitee (开源，国内网站，连接稳定)、极狐 GitLab（私有仓库，本地部署、安全性高）

一、什么是Git

Git 是一个免费开源的分布式版本控制系统。其中，仓库（Repository）被用于记录控制版本的变化，包括何时何人修改何文件的何内容，以便在需要时将文件恢复至特定版本。

Git 的安装下载地址：Git - Downloads。在终端输入命令git -v，如果出现 Git 的版本信息说明安装成功！

注：Windows 版本的 Git 会自动安装 Git Bash（桌面右键鼠标可见）。

三、本地仓库的创建与管理

工作区域和文件状态

Git 仓库中有三个工作区域：工作区、暂存区和本地仓库。

  • 工作区（Working Directory）也称工作目录、本地工作目录。就是在电脑的资源管理器里能实际看到的目录 （ls命令查看）。

  • 暂存区（Staging Area / Index）也称索引，用来临时存放即将提交到 Git 仓库的修改内容，一般在 .git 目录下的 index 中（git ls-files命令查看）。

  • 本地仓库（Local Repository）是 Git 在本地的版本库，仓库信息存储在 .git 这个隐藏目录中。

注：远程仓库（Remote Repository）是托管在远程服务器上的仓库。常用的有 Github、GitLab 和 Gitee。

在工作时，我们在工作区添加修改内容，然后将修改内容临时存到暂存区，修改完成后将所有修改内容提交到本地仓库。

相应地，Git 仓库中的文件也有四种状态：未跟踪、未修改、已修改和已暂存。

未跟踪（Untrack）表示新创建、未被Git管理的文件。

未修改（Unmodified）表示已被管理、但文件内容无变化即未修改的文件。

已修改（Modified）表示修改了但是没有保存到暂存区的文件。

已暂存（Staged）表示修改后已经保存到暂存区的文件。

注：已提交（Committed）表示把暂存区的文件提交到本地仓库后的状态。

创建仓库

创建仓库主要有以下两种方式

  • 本地创建：通过命令 git init [repoName] 将电脑本地需要管理的目录变为 Git 可以管理的仓库。其中仓库名是可选项，如果添加仓库名，就会在当前目录下创建新的仓库名文件作为仓库。

  • 远程克隆：通过命令 git clone xxx.git 从远程服务器上克隆已经存在的仓库。克隆远程仓库的地址有HTTPS和SSH两种方式。HTTPS方式在将本地代码push至远程仓库时，需要验证用户名和密码。SSH方式使用SSH协议，在push时不需要验证用户名，但是首次使用需要配置SSH公钥（必须是项目的拥有者或管理者，否则无法添加 SSH key）。因此我们在克隆别人的Github项目时，一般使用HTTPS方式的仓库地址；管理自己的仓库时，使用SSH方式更方便。

新建/复制 git 本地仓库后，界面不显示分支名称：git config --global --add safe.directory [当前本地仓库地址，如 D:/file]。奇怪但有效~

六、分支

Git 中的分支可以独立存在，并且有自己的提交记录，非常适合团队协作和开发管理。

  • 多个开发人员可以同时在自己的分支上进行开发工作，最后再合并到主线代码库中，提高团队协作效率。
  • 可以设置新分支用于新功能开发或bug修复，且保持主线代码库处于随时可用的稳定状态，不影响其他功能的开发和测试。

基本操作

  • 修改当前分支名称：git branch -M [Name]
  • 查看当前仓库的所有分支：git branch
  • 创建新分支：git branch [branchName]（新分支会继承当前分支上的文件）
  • 切换分支：git switch [branchName] / git checkout [branchName](有歧义，更推荐前者)
  • 将 sourceBranch 合并到 targetBranch：
    • 法1（merge圈）：先切换到 targetBranch，然后 git merge [sourceBranch]
    • 法2（rebase线）：当前分支为 sourceBranch / targetBranch，然后 git rebase [targetBranch / sourceBranch]
  • 查看分支图/提交ID：git log --graph --oneline --decorate --all（可以通过命令alias graph="git log --graph --oneline --decorate --all"为这条命令起代号为“graph”，此后只需要输入“graph”即可）
  • 删除已经完成合并的分支：git branch -d [branchName]
  • 强制删除未完成合并的分支：git branch -D [branchName]
  • 将分支回退到之前的某版本：git checkout -b [branchName] commitID（版本ID可通过VSCode的可视化图形，或git log --graph --oneline --decorate --all找到）
